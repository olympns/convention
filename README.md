Данный раздел содержит конвенцию команды — набор правил, процессов и инженерных стандартов, используемых при работе над проектами.

Конвенция описывает:

- взаимодействие участников команды
- процессы разработки
- работу с задачами и репозиториями
- инженерные практики
- требования к окружению
- работу с git

## Конвенция используется для:

- подключения новых участников
- синхронизации процессов команды
- стандартизации разработки
- снижения количества ошибок и недопонимания
- масштабирования проектов

Документ является живым и может обновляться по мере развития команды.
